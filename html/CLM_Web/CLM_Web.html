View 1



Choose CLM Version

Search Function Name

By choosing a function name (e.g., clm_drv) from a CLM version (e.g., CLM_45_10), one can explore the 3-tier call tree with the selected function being the root node. If any node on the third tier is green, it means the node has called other functions and can be further visualized using another call tree provided by View 2 below.


View 2



Choose CLM Version

Search Function Name

This view has the same functionalities as View 1. The purpose of providing two views is to allow users to: (1) explore call tree structures at different depths in one CLM version; (2) compare call tree structures of the same function between two different CLM versions.


Search Downward



Choose CLM Version

Search Function Name

By choosing a function name from a CLM version, the table on the right panel will return all the variables and functions that have been used in the selected function. The type of each variable is also shown. For example, Subroutine-in variables are the ones identified in the subroutine's signature. Subroutine-out are a subset that was identified to be written to. Global variables are the ones found in the source-code line that adheres to the general pointer assignment syntax.

The number of functions as well as the number of variables for each type is summarized on top of the table.

Tips: Click the head to sort the table.


Search Upward



Choose CLM Version

Search Function / Variable Name


Function Variable

By choosing a variable / function from a CLM version, the table on the right panel will return all the functions that have called the selected variable / function.

If a variable is selected, the type of the variable in each of the functions in the table will be shown. If a function is selected, the type will simply be "Function Call".


Choose CLM Version

Search Subroutine Name

List of Subroutines

Visualizing the structure of CLM submodels facilitates the understanding of CLM components at the micro level. By choosing a particular CLM version, users can search the subroutine name in the textbox (e.g., Photosynthesis) and add all the ones they want to investigate into the list. By clicking Generate Graph, a graph structure that summarizes the interrelationships among all related function calls and variables will be presented. Users can also explore the type of a variable by hovering the mouse over a certain node. The group id of a variable indicates whether its type belongs to Subroutine Explicit, Global or Subroutine Explicit & Global in the modeling context.

To view all possible cateogories of CLM variables, please click Here.

Tips: Drag to move the graph. Scroll the mouse to Zoom in/out.
